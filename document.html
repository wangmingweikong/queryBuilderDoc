使用说明

QueryBuilder 是一个基于可视化复杂SQL拼接的一个插件，主要用来缓解用户使用 PLSQL 或 SQL developer 构建复杂SQL时的可读性差的问题。

QueryBuilder 使用了 Bootstrap 框架以及基于 Bootstrap 的技术，构成了一个具有优雅的界面可视化、功能齐全化、实用化的扁平化的 SQL 动态拼接插件。jQuery QueryBuilder 支持跟随窗口缩放，支持主流浏览器心访问（IE8以上，Chrom,fireFox,Opra）。

如果你想快速了解或使用 jQuery QueryBuilder 插件，下面的使用说明将会对你有很大帮助，我们建议，请仔细阅读说明文档，以免在应用至自己项目时造成不必要的麻烦。

开始使用

QueryBuilder 可以让你快速、简单的加载到本插件，是用户感受到 QueryBuilder 插件的轻便、简洁性；正因为如此，请您按照下面的方法步骤进行使用。

引入依赖

请在你的 HTML 应用程序中引入下列文件，就 QueryBuilder 插件来说，插件核心只有 queryBuilder.css ， jquery.queryBuilder.js ，queryBuilder.method.js ，引用的其他样式或者 javascript 文件是因为 QueryBuilder 插件使用了这些第三方的开源组件，因此，使用者在使用本插件时，必须同时引用罗列出来的这些插件。

<!-- QueryBuilder 核心 CSS 文件 -->
<link rel="stylesheet" href="css/queryBuilder.css">
<!-- Bootstrap 核心 CSS 样式文件 -->
<link rel="stylesheet" href="bootstrap-3.3.5-dist/css/bootstrap.min.css">
<!-- QueryBuilder 默认使用的第三方插件 CSS 样式文件 -->

<link rel="stylesheet" href="css/bootstrap-datetimepicker.css">
<!-- jQuery文件。务必在 QueryBuilder.min.js 之前引入 -->
<script src="js/jquery-1.12.3.min.js"></script>
<!-- 最新的 Bootstrap 核心 JavaScript 文件 -->
<script src="bootstrap-3.3.5-dist/js/bootstrap.min.js"></script>
<!-- QueryBuilder 插件默认使用的第三方插件 js -->
<script src="js/Bootstrap-3-Typeahead.js"></script>
<!-- QueryBuilder 插件自定义方法文件 js，必须引入，自定义方法需要在此定义 -->
<script src="js/queryBuilder.method.js"></script>
<!-- QueryBuilder 插件主文件 -->
<script src="js/jquery.queryBuilder.js"></script>
<!-- QueryBuilder 插件全局变量文件 js，使用时不必引入此文件，仅在调用函数时传入这些参数即可 -->
<script src="js/localPara.js"></script>

准备 QueryBuilder 容器

使用 QueryBuilder 插件前，需首先定义一个容器，建议使用 div 标签作为 QueryBuilder插件的容器. 

<div id="sql_builder" class="mainDiv" ></div>

绑定 QueryBuilder 插件

使用下述js方法绑定加载 QueryBuilder 插件. 

$('#sql_builder').queryBuilder({
   sourceData: sourceData ,
   saveJson: saveJson ,
   propertiesTypes: propertiesTypes ,
   submitButton: 'submit_btn' ,
   SQLButton: 'SQL_btn' ,
   SQL_reset: 'SQL_reset' ,
   SQL_reset: 'SQL_reset' ,
   onPageClicked: function(  entity ,  property ,  type ){
      ...
   },
   onSQLBuild: function(  returnVal ,  type ){
      ...
   }
})

参数说明

在使用过程中，请严格按照下面表中的参数说明方式传参，否则，可能造成组件加载不成功的错误。

参数 默认值 描述
sourceData 必填 要加载的所有的类信息，为具有一定格式的 Json 数据
saveJson null 初始化 QueryBuilder 组件时要显示的结构，为具有一定格式的 Json 数据
propertiesTypes 必填 树形参数对应关系，包含自定义属性类型和方法的对应方法，为具有一定格式的 Json 数据
submitButton 可不填 一个标签的 id 属性值，用来给该 id 树形的组件绑定触发返回构建的 SQL 查询结构的事件
SQLButton 可不填 一个标签的 id 属性值，用来给该 id 树形的组件绑定触发返回构建的 SQL 语句的事件
SQL_reset 可不填 一个标签的 id 属性值，用来给该 id 树形的组件绑定触发重置 SQL 查询结构的事件
onSQLBuild 建议写 此参数对应一个函数，此方法有两个从组件传入的参数，返回给用户构建的 SQL 结构
onPageClicked 必写 此参数对应一个负责组件和后台程序的交互，根据组件传入参数从后台获取数据并返回给组件的函数

submitButton

此参数对应要给标签的 id 属性值，为字符串，当您传入这个标签的 id 属性值后，QueryBuilder 插件可以为这个标签绑定一个 onclick 的事件，当您点击这个标签时，就会触发对应的组件函数，并触发相应的方法。:

例如:您有如下的一个标签

<input id="sql_get" class="form-controll" value="获取SQLl" ></div>

那么您在传入这个参数时应该为以下键值对： 

submitButton:'sql_get',

onSQLBuild

此方法用户接收从 QueryBuilder 组件构建的查询结构或 SQL 语句。为方便用户辨别返回的是哪种值，我们设置了如下两个参数 returnVal 、 type ，分别表示传入的结果或 SQL 语句和传参类型，其中 0 代表 返回的是查询结构，1表示返回的是 SQL 语句

用户可以在这个方法中接收并进行下一步的处理。

onPageClicked

此方法的主要作用是在组件和用户之间数据交流的桥梁，通过这个优化的回调方法，可以实时和数据库进行交流，减轻缓存压力。

这个功能是国外的一些具有 QueryBuilder 类似功能的插件所不能比的（它们通常是仅仅提供一个组件，没有数据交流，因此，必须通过初始化组件时加载所有的肯能出现的数据，这就会造成不必要的缓存压力）。

当使用 jQuery QueryBuilder 组件时，我们常常需要在组件中实时显示后台查询到的数据，比如：用户新建一个查询规则

user.name != '......',

这个不等于的值常常是需要从后台获取，并显示在组件输入框的提示信息中，那么我们就通过这个回调函数来提供一个供用户的后台和组件之间进行数据交流的方式。

在这个回调方法中，组件会向方法传递三个参数 entity(规则的实体类) ，property（规则的属性），type（规则的属性类型）

用户在此方法中根据在 propertiesTypes 定义好的树形规则添加对应的和后台交流方式，然后将从后台获取到的数据以数组的形式返回给组件。

您需严格按照如下格式引用此参数:

   onPageClicked: function(  entity ,  property ,  type ){
      ...
      return [ arrayList ]
   }

返回的数组格式如下：

<!--['当用户输入时显示出来的信息-$-自定义信息，可用于便于用户搜索']，具体信息参考  Bootstrap
                        typeahead -->
['张三-$-zhangshan','李四-$-lisi']
